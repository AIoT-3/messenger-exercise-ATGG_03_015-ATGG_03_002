# 추가 구현 3 - 성능 및 확장성 개선 (Queue 활용)

본 문서는 메신저 시스템의 성능을 최적화하고 확장성을 확보하기 위한 기술적 방안(Queue 활용)을 정의합니다.

## 공통 사항
* 본 문서의 내용은 [기본 메시지 API 문서](message-api-spec.md)에 정의된 시스템 구조를 바탕으로 합니다.

---

## 메시지 큐 (Message Queue) 도입

> 대규모 사용자가 동시에 접속하거나 트래픽이 폭주할 때 서버의 부하를 줄이고 응답성을 높이기 위한 전략입니다.

### 주요 도입 전략

*   **비동기 메시지 처리**: 메시지 수신 시 DB 저장이나 브로드캐스트를 즉시 수행하지 않고 Queue에 쌓아둠으로써 워커 스레드가 순차적으로 처리하게 합니다. 이는 순간적인 트래픽 폭주 시 서버가 다운되는 것을 방지합니다.
*   **Push 알림 분리**: 메시지 전송과 알림(Push) 발송 로직을 분리하여, 메시지 전송 API의 응답 속도를 개선합니다.
