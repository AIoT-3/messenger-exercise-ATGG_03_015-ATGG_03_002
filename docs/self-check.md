# 셀프 체크리스트 (Self-Checklist)

본 문서는 메신저 시스템 개발 완료 후 요구사항 충족 여부를 스스로 확인하기 위한 체크리스트입니다.

## 1. 프로젝트 기본 구성 및 기술 스택
- [x] Maven 멀티 모듈 구조로 구성되어 있는가? (`messenger-common`, `messenger-server`, `messenger-client`)
- [x] Java 21 버전을 사용하는가?
- [ ] Jackson 라이브러리를 사용하여 JSON 직렬화/역직렬화를 처리하는가?
- [ ] TCP 소켓 통신을 기반으로 구현되었는가?

## 2. 공통 규격 및 비기능적 요소
- [ ] **메시지 규격**
    - [ ] Text-based Length-Prefix 방식을 준수하는가? (`message-length: [길이]\n{JSON}`)
    - [ ] 모든 요청의 헤더에 `sessionId`가 포함되어 있으며, 서버에서 이를 검증하는가?
- [ ] **예외 처리**
    - [ ] 모든 실패 응답이 공통 에러 형식(`code`, `message`)을 따르는가?
- [ ] **보안**
    - [ ] 인증된 사용자만 시스템 이용이 가능한가?

## 3. 기본 기능 구현 (Basic Features)
- [ ] **로그인 및 사용자 관리**
    - [ ] `LOGIN`: 로그인 요청 및 성공(sessionId 발급)/실패 응답이 정상적인가?
    - [ ] `LOGOUT`: 로그아웃 요청 및 세션 종료 처리가 정상적인가?
    - [ ] `USER-LIST`: 전체 사용자 목록 및 온라인 상태 조회가 가능한가?
- [ ] **채팅방 관리**
    - [ ] `CHAT-ROOM-CREATE`: 채팅방 생성이 정상적으로 작동하는가?
    - [ ] `CHAT-ROOM-LIST`: 채팅방 목록 조회가 정상적으로 작동하는가?
    - [ ] `CHAT-ROOM-ENTER`: 채팅방 입장 및 현재 참여자 목록 반환이 정상적인가?
    - [ ] `CHAT-ROOM-EXIT`: 채팅방 나가기 처리가 정상적인가?
- [ ] **메시징 기능**
    - [ ] `CHAT-MESSAGE`: 채팅방 내 메시지 전송 및 브로드캐스트가 정상적인가?
    - [ ] `PRIVATE-MESSAGE`: 특정 사용자에 대한 귓속말 전송이 가능한가?
    - [ ] `CHAT-MESSAGE-HISTORY`: 과거 메시지 기록 조회가 가능한가?
    - [ ] 메시지 전송 성공 시 고유한 `messageId`를 반환하는가?

## 4. 추가 구현 사항 (Advanced Features)
- [ ] **추가 구현 1: 파일 전송**
    - [ ] `FILE-TRANSFER`: Base64 인코딩을 통한 파일 전송이 구현되었는가?
    - [ ] 파일 크기 제한(10MB) 초과 시 에러 처리를 하는가?
- [ ] **추가 구현 2: 실시간 알림 (Push)**
    - [ ] `PUSH-NEW-MESSAGE`: 새 메시지 수신 시 실시간 알림을 보내는가?
    - [ ] `PUSH-ROOM-ENTER` / `PUSH-ROOM-EXIT`: 사용자 입장/퇴장 알림이 전달되는가?
- [ ] **추가 구현 3: 성능 개선 (Queue)**
    - [ ] 메시지 저장 및 브로드캐스트 처리를 비동기(Queue 활용)로 처리하는가?
    - [ ] Push 알림 발송 로직이 메인 메시징 로직과 분리되어 있는가?
- [ ] **추가 구현 4: 아키텍처 개선 (Design Patterns)**
    - [ ] **Observer 패턴**: UI와 메시지 수신 로직이 분리되어 있는가?
    - [ ] **Command 패턴**: 채팅 명령어(/list, /exit 등) 처리가 객체화되어 있는가?
    - [ ] **Factory Method 패턴**: 메시지 타입별 객체 생성 로직이 캡슐화되어 있는가?
    - [ ] **Strategy 패턴**: 메시지 타입별 처리 알고리즘이 동적으로 교체 가능한가?
- [ ] **추가 구현 5: NIO 기반 커넥션 관리**
    - [ ] Java NIO를 사용하여 Non-blocking I/O 모델로 구현되었는가?
    - [ ] Selector를 활용하여 대규모 동시 접속을 효율적으로 관리하는가?
